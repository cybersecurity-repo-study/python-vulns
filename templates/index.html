üíâ SQL Injection

Demonstrates how unsanitized user input can be used to manipulate database queries

GET

Vulnerable Endpoint

/sql_vuln?username=<value>
Builds SQL queries using string concatenation. Try payloads like: ' UNION SELECT 1,sqlite_version()--
Example: ?username=alice
Payload: ?username=' UNION SELECT 1,sqlite_version()--
Test Vulnerable
GET

Safe Endpoint

/sql_safe?username=<value>
Uses parameterized queries to prevent SQL injection attacks
Same input, but safely handled with prepared statements
Test Safe

‚ö° Remote Code Execution (RCE)

Shows how insecure subprocess usage can lead to arbitrary command execution

POST

Vulnerable Endpoint

/rce_vuln
Uses shell=True with user input. Dangerous! Use only benign commands like "echo hello"
Body: {"cmd": "echo hello"}
Test Vulnerable
POST

Safe Endpoint

/rce_safe
Validates input and uses argument lists instead of shell=True
Body: {"filename": "."}
Test Safe

üåê Server-Side Request Forgery (SSRF)

Demonstrates how unrestricted URL fetching can be exploited

GET

Vulnerable Endpoint

/fetch_vuln?url=<url>
Fetches any URL provided by user without validation
Example: ?url=https://example.com
Test Vulnerable
GET

Safe Endpoint

/fetch_safe?url=<url>
Uses whitelist and DNS resolution checks to prevent SSRF
Only allows: example.com, httpbin.org
Test Safe
